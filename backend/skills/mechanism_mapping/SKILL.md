# Mechanism Mapping

## Overview

这个 skill 用于处理**机理分析**任务。

它适用于这样的问题：给定一组文献、表征结果、实验现象或初步假设，需要把其中可能存在的结构特征、活性位点变化、中间过程和反应结果组织成一条或多条可审查的机理链条，并明确哪些环节已有证据支持，哪些仍属于推断、类比或待验证内容。

这个 skill 不负责给出唯一正确答案，而是负责输出一个**结构化的机理图谱**，帮助后续实验设计、数据判读、证据补强和写作组织。

---

## When to Use This Skill

当满足以下任一情况时使用本 skill：

- 需要把零散的文献结论、实验现象和个人假设组织成机理链条
- 需要分析某种结构特征或材料变化如何影响活性位点状态
- 需要比较多个候选机理分支，而不是只保留单一路径
- 需要区分哪些环节是证据支持，哪些只是合理推断
- 需要先搭建机理图谱，再决定后续该补什么表征或实验
- 需要从“结构/位点/过程/结果”四层之间建立明确映射关系

---

## When NOT to Use This Skill

以下情况不要使用本 skill：

- 只是做事实检索或资料摘录
- 只是总结单篇文献内容
- 只是描述单张图或单个实验结果
- 只是做排版、润色或汇报页面美化
- 已经有明确原始图谱需要联合判读
  - 这类问题更适合交给图谱判读类 skill
- 主要任务是比较两条反应路径或两类氧化剂
  - 这类问题更适合交给路径比较类 skill
- 任务重点已经转成“把多个 claim、literature note、experiment note 聚合成统一证据链交付物”
  - 这类问题更适合交给证据链聚合类 skill

## Required Inputs

### Minimum Inputs

至少需要以下输入之一的组合：

- 一个明确的机理问题
- 一组相关文献、笔记或摘录
- 一组表征线索、实验现象或初步结论

### Optional Inputs

以下输入会显著提升结果质量：

- 相关图的截图或图号
- 用户当前相信的若干候选解释
- 各类表征结果的简要摘要
- 已知性能变化或对照现象
- 已整理的证据链草稿
- 不同候选路径下的判别性实验想法

### Supported Material Types

- pdf
- md
- figure screenshots
- ppt notes
- literature summary tables
- experiment notes
- raw interpretation drafts

## Expected Outputs

本 skill 应输出以下类型的结构化结果：

- `mechanism_question`
  - 需要回答的核心机理问题
- `branch_map`
  - 候选机理分支列表
- `feature_to_site_mapping`
  - 从结构特征到位点变化的映射
- `site_to_process_mapping`
  - 从位点变化到中间过程或反应结果的映射
- `evidence_status`
  - 每条分支当前的证据状态
- `unresolved_issues`
  - 尚未解决的关键问题
- `validation_suggestions`
  - 最值得优先验证的环节
- `mechanism_narrative_spine`
  - 用 3 到 6 句串起当前最稳的主线叙事

输出形式可以是：

- 机理图谱
- 分支表格
- 逐层映射表
- 证据状态清单
- 待验证问题列表
- 简短机理叙事骨架

## Workflow

### Step 1. 明确机理问题

先把输入重述为一个清晰的问题句。

这个问题句通常应回答：

- 需要解释的核心现象是什么
- 需要连接的关键层次是什么
  - 结构
  - 位点
  - 中间过程
  - 反应结果
- 当前是想做解释、比较，还是假设生成

不要一开始就进入细节，先把问题钉稳。

### Step 2. 规范化候选特征与候选路径

从输入中提取所有可能相关的结构特征、位点状态变化和候选机理路径。

常见对象包括：

- 结构差异
- 配位变化
- 电子结构变化
- 界面效应
- 中间体形成可能性
- 路径选择差异

这一阶段的重点是**保留分支**，而不是过早合并。

### Step 3. 建立“特征 -> 位点 -> 过程 -> 结果”的主链

对每个候选分支，按以下顺序组织：

`结构特征 -> 位点变化 -> 中间过程含义 -> 最终反应结果或路径偏好`

每一层都要写成具体关系，而不是泛泛而谈。

### Step 4. 为每条分支附加证据状态

对每条分支标注证据强度，建议至少分为：

- 直接证据支持
- 间接证据支持
- 类比支持
- 纯推断
- 仍待验证

这一步的目的不是削弱表达，而是防止把推断写成事实。

### Step 5. 比较并排序分支

从两个维度比较每条分支：

- 当前证据支撑强度
- 对后续实验或判读的实用性

允许存在“科学上有吸引力，但当前证据较弱”的分支，只要明确标注即可。

### Step 6. 输出结构化机理图谱

将结果整理为一个可复用的分析产物，通常包括：

- 核心问题
- 机理分支总览
- 逐分支映射链
- 证据状态
- 未解决问题
- 优先验证建议
- 机理叙事骨架

## Boundary with Sibling Skills

使用本 skill 时，注意与相邻 skill 的边界：

- 当重点是**单一体系内部**从结构/位点/过程/结果搭建机理分支时，优先使用本 skill
- 当重点是**两条路径或两类氧化剂的并排比较**时，优先转给路径比较类 skill
- 当重点是**单张谱图或一组谱图的具体判读**时，优先转给图谱联合判读类 skill
- 当重点是**探针、活性物种与证据等级矩阵**时，优先转给活性物种证据矩阵类 skill
- 当重点是**把多个 claim、literature note、experiment note 聚合成统一证据链交付物**时，优先转给证据链聚合类 skill
- 当重点是**把结果组织成汇报或写作结构**时，优先转给结果到汇报组织类 skill

---

## Quality Checks

在输出前检查以下内容：

- [ ] 是否用一句话明确写出了机理问题
- [ ] 是否保留了多个候选分支，而不是过早收敛成单一路径
- [ ] 是否把“结构特征 -> 位点 -> 过程 -> 结果”的链条写清楚
- [ ] 是否区分了证据、类比、推断和待验证内容
- [ ] 是否避免了“协同增强”“电子转移改善”这类空泛表述
- [ ] 是否指出了尚未解决的关键问题
- [ ] 是否给出了后续验证建议
- [ ] 是否产出了可复用的结构化结果，而不只是综述性段落
- [ ] `mechanism_narrative_spine` 是否足够短、稳、可接入下游写作或汇报

---

## Common Failure Modes

使用本 skill 时，尤其要避免以下问题：

- 看到一个顺眼的解释就过早收敛成单一路径
- 把结构变化直接跳写成结果变化，中间缺少位点层和过程层
- 用“协同效应增强”“电子转移改善”代替具体机理关系
- 把类比支持写成直接证据
- 把“能支持某条路径”误写成“足以区分多条路径”
- 只输出讨论感段落，没有分支表、映射链、证据状态等结构化结果
- 给补实验建议时过泛，例如只写“建议补更多表征”

## Example Patterns

### Example Pattern 1 · 从结构变化到路径偏好的机理图谱

#### Typical User Ask
“我现在有几条零散线索：一个是材料结构发生了变化，一个是表征提示位点状态变了，一个是反应表现也不同。请不要直接下结论，先把可能的机理分支列出来，并区分哪些有证据、哪些只是合理推断。”

#### Typical Inputs
- 2 到 5 篇相关文献的摘录或图
- 若干表征现象摘要
- 一组性能或路径差异现象
- 用户当前的候选解释 1 到 3 条
- 可选：已有证据链草稿

#### Expected Output Shape
- `mechanism_question`
- `branch_map`
- `feature_to_site_mapping`
- `site_to_process_mapping`
- `evidence_status`
- `unresolved_issues`
- `validation_suggestions`
- `mechanism_narrative_spine`

#### Boundary Notes
- 本场景适合“先搭机理图谱，再决定补什么证据”
- 如果任务重心转成统一证据链交付，应转给证据链聚合类 skill
- 如果输入主体是单张谱图或单类表征，优先转给图谱判读类 skill

---

### Example Pattern 2 · 从多条候选机理链条反推关键判别实验

#### Typical User Ask
“我现在不想只听一个解释。请先把可能的机理链条分开列出来，再告诉我：同一组已有实验在不同链条下分别支持什么、不支持什么，以及下一步做哪类实验最能把这些路径区分开。”

#### Typical Inputs
- 2 到 4 条候选机理解释
- 一组已有实验现象摘要
- 若干表征结果摘要
- 相关文献中的支持性片段或图号
- 可选：用户已经整理过的初步证据链

#### Expected Output Shape
除了标准机理图谱外，还应特别强调：

- `evidence_status` 中哪些证据真正具有区分性
- 哪些证据只是支持但不具判别力
- `validation_suggestions` 中优先给出**判别性最强**的实验或表征

#### Boundary Notes
- 本场景适合“多路径并存、要靠证据继续收敛”的机理分析
- 如果任务重心已经从“搭图谱”转向“审计和汇总 evidence pack”，应转给证据链聚合类 skill
- 如果重点是两类路径的严格并排比较，应转给路径比较类 skill

## Example Requests

以下请求适合调用本 skill：

- “帮我把这个体系可能的机理分支全部列出来，不要先下结论。”
- “我想知道结构变化可能如何影响活性位点，并进一步改变反应路径。”
- “把这些零散表征和文献假设整理成一条可审查的机理图谱。”
- “先区分哪些环节有证据，哪些只是合理推断。”
- “请把可能路径分开列出，并告诉我下一步做什么最能把它们区分开。”
