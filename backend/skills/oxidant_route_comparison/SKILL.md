# Oxidant Route Comparison

## Overview

这个 skill 用于处理**反应路径比较**任务。

它适用于这样的问题：面对两类氧化剂、两条反应路径，或两组相关机理文献，需要判断它们在活性位点要求、关键基元步骤、证据支撑方式和语言可迁移性上的相同点与差异点，并明确哪些结论可以迁移，哪些只能类比，哪些必须重新证明。

这个 skill 的重点不是做百科式综述，而是产出一个**可审查的比较框架**，防止将一个体系中的机理语言不加区分地照搬到另一个体系中。

---

## When to Use This Skill

当满足以下任一情况时使用本 skill：

- 需要比较两类氧化剂或两条反应路径的机理差异
- 需要判断一个体系中的文献证据能否迁移到另一个体系
- 需要分析不同路径分别对活性位点提出什么要求
- 需要区分“可迁移”“仅可类比”“不可迁移”“必须重证”这几类结论
- 需要为后续写作或实验判读建立边界，避免硬套外部机制
- 需要先搭建对照框架，再决定后续应优先补什么实验或表征
- 需要回答“共享位点要求是否等于共享机制”这一类容易被混淆的问题

---

## When NOT to Use This Skill

以下情况不要使用本 skill：

- 只是总结单一氧化剂或单一路径
- 只是解释一个具体实验结果
- 只是建立反应物种证据矩阵
- 只是做图句锚定或引文核对
- 只是做汇报结构整理
- 问题核心不是“比较两条路径”，而是“构建单一体系机理图谱”
  - 这类问题更适合交给机理分析类 skill
- 任务重点已经转成“把多个 claim、literature note、experiment note 聚合成统一证据链交付物”
  - 这类问题更适合交给证据链聚合类 skill
- 输入主体是单张谱图或单组表征，需要先完成具体判读
  - 这类问题更适合交给图谱联合判读类 skill

## Required Inputs

### Minimum Inputs

至少需要以下输入之一的组合：

- 两个明确的比较对象
- 一个明确的比较问题
- 若干文献、摘录、图或笔记，能够代表两条路径或两个体系

### Optional Inputs

以下输入会显著提升结果质量：

- 代表性文献的关键图号或段落
- 用户想测试可迁移性的具体论断
- 活性位点特征的已有判断
- 需要后续解释的表征或实验结果
- 已整理的比较表草稿
- 用户已经写下的机制表述，但不确定是否越界
- 不同路径下各自最可能成立的位点特征列表

### Supported Material Types

- pdf
- figure screenshots
- md notes
- literature claim tables
- interpretation drafts
- comparison notes

## Expected Outputs

本 skill 应输出以下类型的结构化结果：

- `comparison_question`
  - 本轮比较要回答的核心问题
- `route_summaries`
  - 两条路径的机制摘要
- `site_requirement_matrix`
  - 不同路径对活性位点的要求对照
- `elementary_step_differences`
  - 关键基元步骤的差异
- `transferability_assessment`
  - 证据或机制语言的可迁移性判断
- `non_portable_claims`
  - 不应直接照搬的论断
- `boundary_notes`
  - 写作或解释时应保留的边界提醒
- `validation_priorities`
  - 后续最值得优先验证的内容
- `comparison_narrative_spine`
  - 用 3 到 6 句串起当前最稳的比较主线

输出形式可以是：

- 路径对照表
- 位点要求矩阵
- 可迁移性分级表
- 边界提醒清单
- 验证优先级列表
- 简短比较叙事骨架

## Workflow

### Step 1. 明确比较问题

先把输入整理成一个精确的比较问题。

这个问题通常应回答：

- 比较对象是谁
- 比较维度是什么
- 想判断的是机理差异、位点要求，还是证据可迁移性

示例形式：

`两条反应路径在活性位点要求和关键步骤上有何差异，其中哪些证据可以跨体系迁移？`

不要一开始就急着判断可迁移性，先把“比什么”说清楚。

### Step 2. 用机理语言概括两条路径

分别概括两条路径的核心逻辑，不要只写性能层面的差异。

需要尽量回答：

- 关键活化步骤是什么
- 活性位点在其中扮演什么角色
- 关键反应步骤更偏向哪一类
  - 键裂解
  - 氧转移
  - 电子转移
  - 中间体稳定
  - 自由基释放或抑制
  - 金属中心高价态形成

这一步的目标不是写长综述，而是形成两个**可并排比较的机制摘要**。

### Step 3. 提取不同路径对位点的要求

对每条路径，提取其可能偏好的活性位点特征。

可比较维度包括：

- 配位环境
- 电子贫富特征
- 金属-配体共价性
- 氧化还原灵活性
- 对特定中间态的稳定能力
- 有利于某类基元步骤的局域环境

这一阶段要形成一个并排对照，而不是分散叙述。

### Step 4. 判断证据或机制语言的可迁移性

对每个重要论断进行分级，建议至少分为：

- 可迁移
- 仅可类比
- 不可迁移
- 需要重新验证

判断时要解释原因，而不能只贴标签。

尤其要区分：

- 共享的位点要求
- 共享的中间态语言
- 共享的关键步骤
- 共享的证据逻辑

这四者并不自动等价。

### Step 5. 标出不可直接照搬的部分

显式指出哪些语言或机制结论不应跨体系直接套用。

重点标记：

- 强依赖特定氧化剂化学的步骤
- 强依赖特定位点环境的结论
- 只在原体系中被证明、在新体系中仍未验证的内容
- 看起来相似但证据逻辑并不一致的术语

### Step 6. 输出结构化比较结果

将结果整理成一个可复用的比较框架，通常包括：

- 比较问题
- 两条路径摘要
- 位点要求矩阵
- 关键步骤差异
- 可迁移性分级
- 边界提醒
- 优先验证建议
- 比较叙事骨架

## Boundary with Sibling Skills

使用本 skill 时，注意与相邻 skill 的边界：

- 当重点是**两条路径、两类氧化剂或外部先例与当前体系之间的比较**时，优先使用本 skill
- 当重点是**单一体系内部**从结构/位点/过程/结果搭建机理分支时，优先转给机理分析类 skill
- 当重点是**探针、活性物种与证据等级矩阵**时，优先转给活性物种证据矩阵类 skill
- 当重点是**单张谱图或一组谱图的具体判读**时，优先转给图谱联合判读类 skill
- 当重点是**把多个 claim、literature note、experiment note 聚合成统一证据链交付物**时，优先转给证据链聚合类 skill
- 当重点是**把比较结果组织成汇报或写作结构**时，优先转给结果到汇报组织类 skill

---

## Quality Checks

在输出前检查以下内容：

- [ ] 是否明确写出了比较问题
- [ ] 是否真正比较的是“路径”，而不是只比较性能或结论表面文字
- [ ] 是否并排整理了不同路径对位点的要求
- [ ] 是否区分了“共享位点要求”与“共享关键步骤”
- [ ] 是否对可迁移性进行了分级，而不是默认可借用
- [ ] 是否明确标出不可直接照搬的部分
- [ ] 是否解释了每一类可迁移性判断的理由
- [ ] 是否加入了边界提醒，防止后续写作过度论断
- [ ] 是否给出了优先验证建议
- [ ] `comparison_narrative_spine` 是否足够短、稳、可接入下游写作或汇报

---

## Common Failure Modes

使用本 skill 时，尤其要避免以下问题：

- 把“都需要高活性位点”误写成“机制相同”
- 只比较性能或结果，不比较关键步骤
- 看到外部文献中有熟悉术语就直接迁移
- 不区分“可以借位点要求”和“不能借具体步骤”
- 把类比支持写成已证实迁移
- 可迁移性只给标签，不解释原因
- 只输出比较结论，没有边界提醒和重证位置
- 把整套外部机制整体打包搬进当前体系叙事

---

## Red Flags / Escalation Notes

出现以下情况时，应降低结论强度或提示转交：

- 两个比较对象本身并不处于同一分析层级  
  例如一个是氧化剂，一个是整套材料体系
- 输入只有一篇外部文献，却试图推出整类路径的一般规律
- 当前体系几乎没有自有证据，却想直接继承外部机制主线
- 用户问题实质上是在问某张图、某个谱峰、某条探针结果如何解释
- 用户真正需要的是“给我能写进报告的一版证据链”，而不是路径比较本身

## Example Patterns

### Example Pattern 1 · 比较两条氧化路径对活性位点要求的差异

#### Typical User Ask
“我现在看到两类外部文献路径都在讨论高活性中间状态，但我不想直接照搬。请帮我比较这两条路径到底分别依赖什么样的活性位点，哪些结论可以迁移，哪些只是看起来像，实际上不能直接借来解释当前体系。”

#### Typical Inputs
- 2 到 4 篇代表两条路径的文献摘录
- 每条路径各 1 到 2 个关键机制 claim
- 若干位点特征关键词
- 可选：当前体系的初步表征线索
- 可选：用户已写出的机制表述草稿

#### Expected Output Shape
- `comparison_question`
- `route_summaries`
- `site_requirement_matrix`
- `elementary_step_differences`
- `transferability_assessment`
- `non_portable_claims`
- `boundary_notes`
- `validation_priorities`
- `comparison_narrative_spine`

#### Boundary Notes
- 本场景适合“路径比较 + 可迁移性判断”
- 如果任务重点转成单一体系的机理分支搭建，应转给机理分析类 skill
- 如果任务重点转成多源 evidence 的统一聚合，应转给证据链聚合类 skill

---

### Example Pattern 2 · 审计外部机制先例在当前问题中的可迁移性

#### Typical User Ask
“我手上有一篇外部文献，它给出了很完整的机制叙事，看起来和我现在关注的问题有些相似。但我不想硬套。请把这篇文献里哪些机制语言可以借，哪些只能作为类比，哪些必须在当前问题里重新证明，分开列出来。”

#### Typical Inputs
- 1 篇外部先例文献
- 1 份当前问题的简要描述
- 外部文献中的若干关键 claim、图号或机制语句
- 可选：当前问题已有的表征或实验现象摘要
- 可选：用户已经写好的比较性论述草稿

#### Expected Output Shape
除了标准比较结果外，还应特别强调：

- 逐条审计 claim，而不是整体打包借用
- 明确指出哪些只是“有参考价值”，但还不能升级为当前问题中的主结论
- 在 `boundary_notes` 中给出更克制的写法建议

#### Boundary Notes
- 本场景适合“单篇先例 + 当前问题”的机制迁移性审计
- 如果重点变成具体图或具体实验数据该如何解释，应转给图谱或多证据判读类 skill
- 如果重点变成正式写作中的 claim-evidence 对齐，应转给图句锚定或证据链聚合类 skill

## Example Requests

以下请求适合调用本 skill：

- “这两条路径的关键差异到底在氧化剂本身，还是在活性位点要求上？”
- “哪些文献结论可以借来解释我的体系，哪些只能做参考？”
- “帮我比较这两类氧化路径对活性位点电子结构和配位环境的要求。”
- “我不想硬套外部机制，请把可迁移和不可迁移部分分开。”
- “外部先例到底能解释到哪一层，哪些地方还得重证？”
