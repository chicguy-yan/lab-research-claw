# Example 2 · 审计外部机制先例在当前体系中的可迁移性

## Typical User Ask
“我手上有一篇外部文献，它给出了很完整的机制叙事，看起来和我现在的体系有些相似。但我不想硬套。请把这篇文献里哪些机制语言可以借，哪些只能作为类比，哪些必须在当前体系里重新证明，分开列出来。”

## Typical Inputs
- 1 篇外部先例文献
- 1 份当前体系的简要描述或目标问题
- 外部文献中的若干关键 claim、图号或机制语句
- 可选：当前体系已有的表征或实验现象摘要
- 可选：用户已经写好的比较性论述草稿

## Expected Output Shape

### comparison_question
一句话说明当前任务不是“是否完全相同”，而是“外部先例的哪些机制要素具有可迁移性”。

### route_summaries
分别概括：
- 外部先例中机制的核心逻辑
- 当前体系真正关心的反应路径或位点问题

### transferability_assessment
逐条审计外部文献中的关键 claim，并分为：
- 可迁移
- 仅可类比
- 不可迁移
- 需要重新验证

### non_portable_claims
列出最容易被误搬进当前体系、但其实不应直接照搬的结论或术语

### boundary_notes
说明在当前体系中可以如何更克制地表述这些外部先例

### validation_priorities
指出当前体系需要补哪些证据，才能把“类比”升级为“较稳的可迁移论断”

### comparison_narrative_spine
用 3 到 5 句写出最稳妥的叙事方式：
- 外部先例在哪个层面有参考价值
- 哪些部分只是启发而不是证据
- 下一步如何补证

## Common Failure Modes
- 把“外部文献很像”当成“当前体系已经被证明”
- 没有逐条审计 claim，而是整体打包借用
- 只说“可以参考”，但不说参考到哪一层
- 把外部机制术语直接搬进当前体系的主结论
- 没有指出当前体系最关键的重证位置

## Boundary Notes
- 本 example 适合“单篇先例 + 当前体系”的机制迁移性审计
- 如果重点是两类路径的系统性对比，应转给更标准的路径比较输出
- 如果重点变成具体图或具体实验数据该如何解释，应转给图谱或多证据判读类 skill
- 如果重点变成正式写作中的 claim-evidence 对齐，应转给 `figure_claim_anchoring` 或 `mechanism_evidence_chain`
