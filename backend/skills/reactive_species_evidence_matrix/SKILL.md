# Reactive Species Evidence Matrix

## Overview

这个 skill 用于处理**活性物种证据整理与分级**任务。

它适用于这样的问题：用户手里有一组与活性物种相关的实验结果，例如探针反应、淬灭实验、EPR 信号、活性变化、对照实验或文献先例，希望判断：

- 每一种证据分别支持什么
- 哪些证据只是排除法
- 哪些证据可以视为较强支持
- 哪些证据并不具区分性
- 多种证据组合起来后，最多能把结论推进到哪一层

这个 skill 的重点不是罗列“某种探针常用于检测什么”，而是输出一个**可审查的证据矩阵**，帮助后续机制判断、写作和补实验设计。

---

## When to Use This Skill

当满足以下任一情况时使用本 skill：

- 需要判断某个探针、淬灭剂或谱学信号到底能证明什么
- 需要区分正证据、旁证、排除性证据和非区分性证据
- 需要把多种活性物种相关实验整理成统一证据框架
- 需要回答“这些结果能支持到哪一层，不能支持到哪一层”
- 需要比较不同活性物种假设在当前证据下各自的支持强弱
- 需要决定下一步做什么实验最能提升证据区分度
- 需要把活性物种相关结果转换成后续机理分析或证据链聚合可复用的中间结果

---

## When NOT to Use This Skill

以下情况不要使用本 skill：

- 只是解释单一实验结果，没有活性物种证据比较需求
- 主要任务是对一张或一组谱图做具体峰位判读
  - 这类问题更适合交给图谱联合判读类 skill
- 主要任务是搭建单一体系的完整机理分支图谱
  - 这类问题更适合交给机理分析类 skill
- 主要任务是比较两条反应路径或两类氧化剂的整体机制差异
  - 这类问题更适合交给路径比较类 skill
- 主要任务是把多个实验、文献和推理聚合成正式证据链交付物
  - 这类问题更适合交给证据链聚合类 skill
- 主要任务是汇报结构整理、图页映射或写作提纲组织

## Required Inputs

### Minimum Inputs

至少需要以下输入之一的组合：

- 一组探针或淬灭实验结果
- 一组 EPR 或其他活性物种相关信号
- 一组活性变化结果，用于比较不同处理条件
- 一组关于活性物种的候选假设
- 一组文献 claim 或参考归属，等待审计其证据等级

### Optional Inputs

以下输入会显著提升结果质量：

- 每个探针或淬灭剂的具体实验现象
- 不同条件下的活性变化幅度
- 对照实验结果
- 文献中对这些探针/淬灭剂/信号的常见解释
- 用户当前最关注的物种假设
- 用户已经写过的一版机制论述
- 用户希望区分的几种 competing explanations

### Supported Material Types

- pdf
- figure screenshots
- spectra images
- md notes
- experiment notes
- interpretation drafts
- literature claim tables
- activity summaries

## Expected Outputs

本 skill 应输出以下类型的结构化结果：

- `evidence_question`
  - 本轮活性物种证据整理真正要回答的问题
- `candidate_species_or_pathways`
  - 当前待比较的活性物种或路径假设
- `evidence_items`
  - 当前纳入分析的证据条目
- `evidence_classification`
  - 每条证据属于正证据、旁证、排除性证据、非区分性证据中的哪一类
- `support_matrix`
  - 证据条目与候选物种/路径之间的支持关系矩阵
- `what_each_can_prove`
  - 每种证据最多能证明什么
- `what_each_cannot_prove`
  - 每种证据不能单独证明什么
- `non_discriminating_evidence`
  - 看似有信息量、但不足以区分候选解释的证据
- `evidence_gaps`
  - 当前关键缺口
- `next_best_checks`
  - 最值得优先补的判别性实验或验证动作
- `evidence_narrative_spine`
  - 用 3 到 6 句写出当前最稳的证据叙事骨架

输出形式可以是：

- 证据等级矩阵
- 证据条目清单
- 能证明/不能证明清单
- 缺口与补实验优先级列表
- 简短证据叙事骨架

## Workflow

### Step 1. 明确证据整理问题

先把输入重述为一个清晰的问题句。

这个问题句通常应回答：

- 当前要比较的是哪几类活性物种或路径假设
- 用户真正困惑的是：
  - 某个探针能否作为直接证据
  - 某组淬灭结果到底在排除什么
  - 某个 EPR 信号是否足够支持某物种
  - 多种证据放在一起时能否得出较稳结论

不要一开始就给结论，先把“这轮证据整理到底要解决什么”说清楚。

### Step 2. 列出候选物种或候选路径

把当前待比较的对象显式列出来。

这些对象可以是：

- 不同活性物种
- 不同反应路径
- 不同中间态假设
- 不同主导机制解释

这一阶段的重点是保留并列候选，而不是默认只有一个正确答案。

### Step 3. 逐条整理证据项

把当前已有实验或文献条目逐条列出来，而不是直接把它们打包成“整体支持”。

常见证据项包括：

- 探针反应现象
- 淬灭后活性变化
- EPR 或其他谱学信号
- 对照样变化
- 特定条件下的选择性变化
- 文献先例中的对应证据逻辑

这一阶段先完成“有哪些证据”，不要急着判断它们的等级。

### Step 4. 给每条证据分类

对每条证据至少分为以下几类之一：

- 正证据  
  对某候选解释具有较直接的支持意义
- 旁证  
  与某解释一致，但不能单独完成证明
- 排除性证据  
  更主要地用于削弱或排除某些解释
- 非区分性证据  
  看似有信息量，但对多个解释都成立，无法真正区分

这一阶段的目标，是把“有信息”与“能区分”分开。

### Step 5. 写清楚每条证据的支持上限

对每条证据继续往前一步，明确回答：

- 这条证据最多能证明什么
- 它不能单独证明什么
- 它更适合支持哪一层判断
  - 是否存在某类物种
  - 是否存在某类路径倾向
  - 是否只是提示趋势
  - 是否只能作为排除某种解释的辅助依据

避免把“与某假设一致”写成“已经证明该假设成立”。

### Step 6. 构建支持矩阵并识别非区分性证据

将所有证据条目与所有候选物种/路径并排对照，形成支持矩阵。

矩阵至少应回答：

- 哪些证据支持候选 A
- 哪些证据只弱支持候选 B
- 哪些证据对多个候选都成立
- 哪些证据根本不具判别力

这一阶段要特别找出：

- 看似关键、其实不具区分性的证据
- 真正能提高区分度的证据空位

### Step 7. 输出证据矩阵与下一步建议

将结果整理成一个可复用的证据框架，通常包括：

- 证据整理问题
- 候选物种/路径列表
- 证据分类
- 支持矩阵
- 能证明 / 不能证明清单
- 非区分性证据
- 缺口
- 最值得优先补的检查项
- 证据叙事骨架

## Boundary with Sibling Skills

使用本 skill 时，注意与相邻 skill 的边界：

- 当重点是**探针、淬灭、EPR、活性变化等证据的等级整理与区分能力判断**时，优先使用本 skill
- 当重点是**某张谱图或某组谱图本身的具体判读**时，优先转给图谱联合判读类 skill
- 当重点是**单一体系内部从结构/位点/过程/结果搭建完整机理分支**时，优先转给机理分析类 skill
- 当重点是**两条路径或两类氧化剂的整体机制差异与可迁移性比较**时，优先转给路径比较类 skill
- 当重点是**把所有证据、文献和实验统一整理成正式证据链交付物**时，优先转给证据链聚合类 skill
- 当重点是**把这些结果排成汇报或写作结构**时，优先转给结果组织类 skill

---

## Quality Checks

在输出前检查以下内容：

- [ ] 是否明确写出了本轮证据整理问题
- [ ] 是否显式列出了候选物种或路径，而不是默认只有一个解释
- [ ] 是否逐条列出证据项，而不是整体打包
- [ ] 是否区分了正证据、旁证、排除性证据和非区分性证据
- [ ] 是否明确说明每条证据“最多能证明什么”
- [ ] 是否明确说明每条证据“不能单独证明什么”
- [ ] 是否找出了非区分性证据
- [ ] 是否指出了关键缺口和最优先补的检查项
- [ ] `evidence_narrative_spine` 是否足够克制、可接入后续机理分析或证据链聚合

---

## Common Failure Modes

使用本 skill 时，尤其要避免以下问题：

- 把探针或淬灭实验的常见用途直接当成当前体系中的直接证明
- 把“活性下降”自动写成某类物种被证实存在
- 把 EPR 信号一出现就直接升级为主机制已定
- 没有区分“支持”和“能区分”
- 把多个弱证据简单叠加，就写成强结论
- 只说“这些结果支持某物种”，不说支持到了哪一层
- 只列证据，不指出哪些其实不具判别力
- 补实验建议过泛，例如“再做更多淬灭实验”

---

## Red Flags / Escalation Notes

出现以下情况时，应降低结论强度或提示转交：

- 输入只有单一证据项，却试图推出完整活性物种主线
- 用户真正想问的是某张谱图或某个峰怎么解释
- 当前证据严重依赖外部文献常见归属，但没有给出当前体系的对照或补强信息
- 多条证据彼此矛盾，但用户仍希望直接写成确定主线
- 用户真正需要的不是证据矩阵，而是正式 claim-evidence pack 或汇报结构

## Example Patterns

### Example Pattern 1 · 整理不同探针/淬灭结果分别能证明什么

#### Typical User Ask
“我这里有几种探针和淬灭实验结果，但我不确定它们到底分别能证明什么。请把这些证据拆开整理：哪些算正证据，哪些只是排除法，哪些其实不能单独支持我想说的结论。”

#### Typical Inputs
- 若干探针或淬灭实验结果
- 各实验对应的活性变化现象
- 可选：文献中常见的归属说法
- 可选：用户当前最关心的活性物种假设

#### Expected Output Shape
- `evidence_question`
- `candidate_species_or_pathways`
- `evidence_items`
- `evidence_classification`
- `support_matrix`
- `what_each_can_prove`
- `what_each_cannot_prove`
- `non_discriminating_evidence`
- `evidence_gaps`
- `next_best_checks`
- `evidence_narrative_spine`

#### Boundary Notes
- 本场景适合“先把每种证据的证明力拆清楚”
- 如果后续要把这些证据接到完整机理分支，应转给机理分析类 skill
- 如果后续要把它们聚合成正式 evidence pack，应转给证据链聚合类 skill

---

### Example Pattern 2 · 把 probe / EPR / activity 放在一起，但避免越级成完整机制

#### Typical User Ask
“我这里不止一种证据，有探针结果，也有 EPR 和活性变化。请帮我把它们放在一起看：哪些地方互相加强，哪些只是平行提示，哪些还不能直接写成主机制。尤其请把能说和不能说分开。”

#### Typical Inputs
- 一组 probe 或 scavenger 实验
- 一组 EPR 或其他活性物种相关信号
- 一组活性变化结果
- 可选：用户已有的一版机制论述草稿
- 可选：待比较的两到三种活性物种或路径假设

#### Expected Output Shape
除了标准证据矩阵外，还应特别强调：

- 哪些证据组合后可以提升稳健性
- 哪些证据只是方向一致，但仍不足以完成证明
- 哪些候选解释被削弱了，哪些仍并存
- 哪些“看似支持”的证据其实不具区分性

#### Boundary Notes
- 本场景适合“多种活性物种相关证据的联合审计”
- 如果重点转成谱图本身如何判读，应转给图谱联合判读类 skill
- 如果重点转成“这些证据最终支持哪条机理分支”，应转给机理分析类 skill
- 如果重点转成正式写作中的 claim-evidence 对齐，应转给证据链聚合类 skill

## Example Requests

以下请求适合调用本 skill：

- “这个探针结果到底能不能算直接证据？”
- “这些淬灭实验分别在排除什么、支持什么？”
- “把 probe、EPR 和活性变化一起放进证据矩阵里看。”
- “哪些结果能说，哪些结果其实不能直接写进主结论？”
- “请帮我找出这些活性物种证据里，哪些真正有区分性。”
