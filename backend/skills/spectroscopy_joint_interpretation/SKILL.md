# Spectroscopy Joint Interpretation

## Overview

这个 skill 用于处理**谱图联合判读**任务。

它适用于这样的问题：用户给出一张或一组谱图、谱峰信息、边位变化、肩峰、配位壳层特征、拟合结果或相关文献对照，希望完成基础读图、暂定归属、支持范围判断，以及与位点状态或机理含义之间的谨慎连接。

这个 skill 不负责把谱图直接升级为完整机制结论，而是负责输出一个**结构化的读图结果**：

- 看到了什么特征
- 这些特征可能意味着什么
- 这些特征最多能支持到哪一层
- 哪些地方还只能作为 tentative 提示
- 还缺什么信息才能进一步收敛

它适合多种谱学或表征类型，包括但不限于：

- XANES
- EXAFS
- WT
- XPS
- EPR
- Raman
- UV-Vis
- 其他具有峰位、边位、壳层或特征信号的谱图

---

## When to Use This Skill

当满足以下任一情况时使用本 skill：

- 需要对一张或一组谱图做基础读图和结构化判读
- 需要标出峰、边、肩峰、配位壳层或其他关键特征
- 需要判断某个谱图特征支持什么结论，以及它的支持上限是什么
- 需要把多个谱图结果联合起来，形成更稳的位点状态判断
- 需要区分哪些谱图结论可以视为较稳的 `confirmed`，哪些只能是 `tentative`
- 需要把谱图读图结果转化为后续实验、写作或机制分析可复用的读图笔记
- 需要先完成谱图层判读，再决定是否进入机理分析或证据链聚合

---

## When NOT to Use This Skill

以下情况不要使用本 skill：

- 只是总结单篇文献整体内容
- 只是比较两条反应路径或两类氧化剂
  - 这类问题更适合交给路径比较类 skill
- 主要任务是搭建单一体系的完整机理分支图谱
  - 这类问题更适合交给机理分析类 skill
- 主要任务是把多个 claim、literature note、experiment note 聚合成统一证据链交付物
  - 这类问题更适合交给证据链聚合类 skill
- 主要任务是做页面组织、汇报结构或写作提纲
  - 这类问题更适合交给结果组织类 skill
- 输入根本不是谱图、峰表、边位信息、拟合结果或相关判读材料

## Required Inputs

### Minimum Inputs

至少需要以下输入之一的组合：

- 一张谱图或一组谱图
- 峰位、边位、肩峰、壳层或拟合特征的描述
- 一组待判断的谱学观察现象
- 与谱图相关的参考图、参考文献或已有归属说法

### Optional Inputs

以下输入会显著提升结果质量：

- 峰位数值及误差范围
- 原始图截图或数据导出
- 对照样之间的谱图变化
- 拟合参数或配位数变化
- 参考文献中的对应图号或已知归属
- 用户当前关注的位点状态或目标问题
- 其他互补谱学结果，用于联合判读

### Supported Material Types

- pdf
- figure screenshots
- spectra images
- peak tables
- fitting summaries
- md notes
- interpretation drafts
- ppt notes

## Expected Outputs

本 skill 应输出以下类型的结构化结果：

- `spectroscopy_question`
  - 本轮谱图判读真正要回答的问题
- `observed_features`
  - 观察到的谱图特征列表
- `provisional_assignments`
  - 每个特征的暂定归属
- `support_scope`
  - 每个特征最多支持到哪一层结论
- `limitations`
  - 每个归属或结论的局限性
- `confirmed_vs_tentative`
  - 区分相对稳的判断与仅提示性的判断
- `cross_spectra_links`
  - 多种谱学结果之间能否互相增强支持
- `missing_information`
  - 当前还缺哪些信息
- `next_checks`
  - 下一步最值得补的判读或表征
- `interpretation_narrative_spine`
  - 用 3 到 6 句总结当前最稳的读图叙事

输出形式可以是：

- 读图笔记
- 特征表
- 特征 → 归属 → 支持范围表
- confirmed / tentative 清单
- 局限性清单
- 下步检查建议

## Workflow

### Step 1. 明确谱图判读问题

先把输入重述为一个清晰的问题句。

这个问题句通常应回答：

- 当前要判读的是哪类谱图
- 用户最关心的到底是：
  - 峰怎么标
  - 特征怎么归属
  - 对照样之间差异说明什么
  - 哪些结论可以说
  - 哪些地方还不能下定论

不要一开始就直接归属，先明确本轮读图的目标。

### Step 2. 识别谱学类型与关键特征

先识别这是哪类谱学或表征，再列出可观察到的关键特征。

常见特征包括：

- 峰位
- 边位
- 白线强度
- 肩峰
- 壳层位置
- 配位数变化
- 峰宽变化
- 拟合差异
- 信号增强或减弱
- 有无某类特征峰

这一阶段只做“看到了什么”，不要急着把所有特征升级成结论。

### Step 3. 给出暂定归属

对每个观察特征，给出合理的暂定归属。

每条归属尽量写成：

`观察特征 -> 可能归属`

例如：

- 峰位移动 -> 可能对应某类电子环境变化
- 壳层变化 -> 可能对应配位环境变化
- 新肩峰出现 -> 可能提示新物种或新局域结构
- 信号减弱/增强 -> 可能提示相对含量或相对贡献变化

这里必须保留“暂定”语气，不要把常见归属写成自动成立的事实。

### Step 4. 标明支持范围

对每个归属继续往前一步，但只写到谱图真正能支持的上限。

建议至少区分：

- 直接支持  
  谱图特征本身已经相当直接地支持某一层判断
- 间接支持  
  该特征与某判断一致，但不能单独完成证明
- 仅提示  
  只能提示趋势或方向，不能单独下结论

这一阶段的目标，是防止把谱图结果越级写成完整机理结论。

### Step 5. 区分 confirmed 与 tentative

将所有归属和判断按稳健性分层。

建议至少区分：

- `confirmed`
  - 当前已有较稳的多重支持
- `tentative`
  - 当前仅能作为提示或弱支持
- `not_supported_enough`
  - 当前证据不足，不应写入主结论

如果有多种谱学结果互相支撑，可以在这里提升部分判断的稳健性，但必须说明为什么。

### Step 6. 联合多谱学结果

如果输入中包含多种谱学结果，尝试建立它们之间的互补关系。

例如：

- 一类谱图更偏电子结构信息
- 一类谱图更偏配位环境信息
- 一类谱图更偏活性物种或局域信号提示

这一步的重点是：

- 哪些结果互相加强
- 哪些结果只是平行出现
- 哪些结果之间其实还存在张力或不一致

不要为了“讲顺”而强行合并不一致信息。

### Step 7. 输出结构化读图结果

将结果整理为一个可复用的读图笔记，通常包括：

- 判读问题
- 观察到的特征
- 暂定归属
- 支持范围
- confirmed / tentative 分层
- 局限性
- 缺失信息
- 下步检查建议
- 简短叙事骨架

## Boundary with Sibling Skills

使用本 skill 时，注意与相邻 skill 的边界：

- 当重点是**一张或一组谱图本身如何判读**时，优先使用本 skill
- 当重点是**单一体系内部从结构/位点/过程/结果搭建完整机理分支**时，优先转给机理分析类 skill
- 当重点是**两条路径或两类氧化剂的并排比较**时，优先转给路径比较类 skill
- 当重点是**探针、活性物种与证据等级矩阵**时，优先转给活性物种证据矩阵类 skill
- 当重点是**把谱图、实验、文献统一聚合成正式证据链交付物**时，优先转给证据链聚合类 skill
- 当重点是**把谱图结果组织进汇报页面或写作结构**时，优先转给结果组织类 skill

---

## Quality Checks

在输出前检查以下内容：

- [ ] 是否明确写出了本轮谱图判读问题
- [ ] 是否先列“观察特征”，再做归属，而不是直接跳结论
- [ ] 每个归属是否写成“暂定归属”，而不是自动成立的事实
- [ ] 是否区分了直接支持、间接支持和仅提示
- [ ] 是否区分了 `confirmed` 与 `tentative`
- [ ] 是否写明了局限性和缺失信息
- [ ] 如果有多谱学联合判读，是否说明了它们是互补、平行还是存在张力
- [ ] 是否避免把谱图结果越级写成完整机理结论
- [ ] `interpretation_narrative_spine` 是否足够短、稳、可接入下游机制分析或写作

## Common Failure Modes

使用本 skill 时，尤其要避免以下问题：

- 只看到一个峰位变化，就直接写成某种位点或机理已经被证明
- 峰位或边位移动没有给出对照对象，就开始赋予明确含义
- 把文献里的常见归属直接照搬到当前谱图，而不说明其局限
- 把“与某结论一致”写成“直接证明了某结论”
- 多种谱学结果之间有不一致，却为了叙事顺滑强行合并
- 只输出一段讨论文字，没有结构化特征表和 confirmed / tentative 分层
- 用“谱图支持了机理”一笔带过，没有说明支持到了哪一层

---

## Red Flags / Escalation Notes

出现以下情况时，应降低结论强度或提示转交：

- 输入只有模糊截图，没有峰位、对照或基本上下文
- 用户试图从单一谱图直接推出完整机理闭环
- 核心问题其实是“该不该比较两条路径”，而不是“这张谱图怎么看”
- 当前谱图归属高度依赖外部参考，但没有给出可比文献或对照样
- 不同谱学结果之间明显矛盾，但用户仍希望直接写成确定主线
- 用户真正需要的是正式证据链或汇报结构，而不是读图本身

## Example Patterns

### Example Pattern 1 · 从峰位/边位变化到谨慎的位点状态判断

#### Typical User Ask
“我这里有一组谱图，对照样之间峰位或边位有变化。请先帮我把这些特征标出来，再告诉我这些变化最多能说明到哪一层，哪些判断可以写得稳一点，哪些只能作为 tentative 提示。”

#### Typical Inputs
- 一组对照样谱图
- 峰位、边位或强度变化的描述
- 可选：参考文献图号或对照图
- 可选：用户当前最关心的位点状态问题

#### Expected Output Shape
- `spectroscopy_question`
- `observed_features`
- `provisional_assignments`
- `support_scope`
- `limitations`
- `confirmed_vs_tentative`
- `missing_information`
- `next_checks`
- `interpretation_narrative_spine`

#### Boundary Notes
- 本场景适合“先把图读清楚，再决定能否上升到位点判断”
- 如果后续要把位点判断组织成完整机理图谱，应转给机理分析类 skill
- 如果后续要把多源结果聚合成正式证据链，应转给证据链聚合类 skill

---

### Example Pattern 2 · 联合多种谱学结果，但避免越级到完整机制

#### Typical User Ask
“我这里不止一种谱图。请帮我把这些结果放在一起看：哪些地方互相支持，哪些地方只是平行提示，哪些地方还不能直接写成机制结论。尤其请帮我区分 confirmed 和 tentative。”

#### Typical Inputs
- 两种或以上谱学结果
- 每种谱学的关键特征摘要
- 可选：已有的位点或中间态猜想
- 可选：用户自己写过的一版解释草稿

#### Expected Output Shape
除了标准读图输出外，还应特别强调：

- `cross_spectra_links`
  - 哪些谱学之间形成互补支撑
  - 哪些只是方向一致但不构成强证明
  - 哪些存在张力或待解释的不一致
- `confirmed_vs_tentative`
  - 哪些结论因为多谱学互证而更稳
  - 哪些仍不能升级

#### Boundary Notes
- 本场景适合“多谱学联合判读，但仍停留在谱图层”
- 如果重点转成“这些谱图共同支持哪条机理分支”，应转给机理分析类 skill
- 如果重点转成“这些结果能否迁移到另一条路径或外部文献”，应转给路径比较类 skill

## Example Requests

以下请求适合调用本 skill：

- “这张图里峰怎么标，哪些特征值得记？”
- “这组对照样的边位变化最多能说明到哪一层？”
- “这些谱图能不能支持位点状态发生了变化？”
- “请帮我区分哪些结论可以写得比较稳，哪些只能是 tentative。”
- “把这几种谱学结果放在一起看，但不要直接跳成完整机制。”
