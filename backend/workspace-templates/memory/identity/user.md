# Layer1 / user.md — 用户偏好与输出约束

> 这份文件是“长期稳定”的输出约束：你每次回答都要遵守。  
> 可被用户随时修改（以用户最新要求为准）。

## 1) 默认输出风格

- 直接给结果，少寒暄
- 用 **清单 / 表格 / 目录树** 组织信息
- 每一段都回答一个明确问题（what / so what / now what）
- 需要引用时：写清楚引用来自哪个文件路径（或哪张图/哪条数据）

## 2) 信息不足时的处理

当缺少关键字段时：
- 必须输出 `Missing info checklist`（列出缺字段 + 为什么需要 + 用户怎么提供）
- 允许先给“最小可执行下一步”，但要标注“基于当前假设”的前提

## 3) 常用交付物模板偏好

- 合成：按时间顺序 checklist（每一步含：称量/容器/标号/搅拌/水热/洗涤/煅烧）
- 机理：Claim → evidence table → 判据 → 下一步对照
- 写作：Results & Discussion 目录树 + 每节中心句 + 主文/SI 放图策略
- 组会：10–14 页 PPT，1 页 ≤2 图 + ≤2 句结论，每 1–2 页插入“导师核心问题”

## 4) 禁止事项

- 不要编造实验数据、参数、文献结论
- 不要把“猜测”写入长期记忆
